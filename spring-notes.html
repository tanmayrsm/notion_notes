🔒 Spring Security
💡
@EnableWebSecurity registers DelegatingFilterProxy into Tomcat — every HTTP request passes through the filter chain before reaching your controller.
Java 
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            .addFilterBefore(jwtFilter(), UsernamePasswordAuthenticationFilter.class)
            .build();
    }
}
⚠️
@EnableWebSecurity does NOT create rules — it just registers the chain. Rules live in your SecurityFilterChain bean.
⚙️ Spring Batch
💡
Chunk-oriented: Job → Step → read N → process → write (one transaction per chunk). Failed jobs restart from last checkpoint stored in JobRepository.
Java 
@Bean
public Step step1(StepBuilderFactory steps) {
    return steps.get("step1")
        .<User, User>chunk(10)
        .reader(flatFileItemReader())
        .processor(userProcessor())   // return null to skip item
        .writer(jpaItemWriter())
        .faultTolerant()
        .skip(Exception.class).skipLimit(5)
        .build();
}
🔌 Dependency Injection
Java 
// ✓ Constructor injection — preferred
@Service
public class OrderService {
    private final PaymentService payment;
    private final InventoryService inventory;

    // No @Autowired needed in Spring 4.3+ with single constructor
    public OrderService(PaymentService payment, InventoryService inventory) {
        this.payment = payment;
        this.inventory = inventory;
    }
}
Bean scopes
singleton — one instance for whole app (default)
prototype — new instance every getBean()
request — one per HTTP request (web only)
session — one per HTTP session (web only)
🔄 Multithreading
Java 
// Always configure a custom executor — default creates new thread per call!
@Configuration
@EnableAsync
public class AsyncConfig {
    @Bean
    public Executor taskExecutor() {
        ThreadPoolTaskExecutor exec = new ThreadPoolTaskExecutor();
        exec.setCorePoolSize(4);
        exec.setMaxPoolSize(16);
        exec.setQueueCapacity(100);
        exec.setThreadNamePrefix("async-");
        exec.initialize();
        return exec;
    }
}

@Service
public class NotificationService {
    @Async
    public CompletableFuture<String> send(String to) {
        // Runs in thread pool, not caller's thread
        return CompletableFuture.completedFuture("sent");
    }
}

🏭 BeanFactory vs ApplicationContext
♻️ Bean Lifecycle
⚠️
@PostConstruct / @PreDestroy are JSR-250 — work in both containers but only auto-register in ApplicationContext via CommonAnnotationBeanPostProcessor.
Java 
@Component
public class CacheService {
    private final DataSource dataSource;

    public CacheService(DataSource dataSource) {
        this.dataSource = dataSource;
        // ❌ don't use dataSource here — Spring hasn't verified it yet
    }

    @PostConstruct
    public void init() {
        // ✓ Safe — all deps injected, all BPPs have run
        cache.loadFromDatabase(dataSource);
    }

    @PreDestroy
    public void cleanup() {
        cache.flush();
        dataSource.close(); // called on context.close() or app shutdown
    }
}
🏆 Injection Priority
1
@Qualifier — most explicit, always wins
Names exact bean at injection site. Overrides @Primary. Use when you always need a specific impl.
@Qualifier("paypalPayment")
2
@Primary — default winner
No annotation needed at injection site. One bean wins when multiple candidates exist.
3
Variable name matching — implicit fallback
Spring matches field name to bean name. Fragile — rename variable and it breaks silently.
4
@Order — List<T> sequence ONLY
Does NOT affect which bean wins single-bean injection. Controls order in List<SomeBean>. Common interview trap.
5
@ConditionalOnMissingBean — Spring Boot auto-config
Library default skipped entirely if you define your own bean. Powers all Spring Boot starters.
Java 
@Primary @Bean public PaymentService stripePayment() { ... }
@Bean         public PaymentService paypalPayment()  { ... }

@Autowired PaymentService payment;               // → stripe  (@Primary)
@Autowired @Qualifier("paypalPayment") PaymentService p2;  // → paypal  (explicit)
@Autowired PaymentService paypalPayment;         // → paypal  (name match)
@Autowired List<PaymentService> all;            // → both, @Order controls sequence
